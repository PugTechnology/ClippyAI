## Field Descriptions:
- approved: boolean - Initial AI recommendation 
- score: integer 1-10 - Overall code quality score
- positives: array - Good things about the code
- issues: array - Problems that MUST be fixed by the Coder
- suggestions: array - Optional improvements
- project_compliance: boolean - Does it follow project rules? (Must be false if README is modified or JOURNAL is missing)
- security_ok: boolean - No security vulnerabilities detected
- verdict: "APPROVE" | "REQUEST_CHANGES" | "REJECT"
- labels: array - Suggested PR labels: ["bug", "enhancement", "documentation", "security", "refactor", "performance", "breaking-change"]

## Scoring Guide:
- 9-10: Excellent - Production ready, well documented, tested
- 7-8: Good - Minor issues, safe to merge with suggestions
- 5-6: Acceptable - Needs some improvements before merge
- 3-4: Poor - Significant issues, needs rework
- 1-2: Critical - Major problems, security issues, rule violations, reject
